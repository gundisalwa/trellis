The catalog, made readable

Fourteen invariants. Why each earns its place.

These are the load-bearing properties Trellis keeps true no matter how fast your agents move. For each: what it is, why it matters, and the same situation with it and without. Everything else about your process, Trellis leaves alone.

with the invariant without it Each example is tagged by the layer it shows up in.

A Structural gate

A four-point admission check: does the process even have the shape to be governed? If it lacks one, Trellis says so loudly.

A1Directional flow

One-way stages of decreasing ambiguity (research → decisions → contracts → implementation → validation); downstream never consumes a draft.

Why: agents always build on settled ground — ambiguity only decreases, so no one codes against a spec that is still moving.

With

specimplementation reads an approved spec; a ratified doc never depends on a draft.

researcha synthesis note cites only ratified findings, not drafts still under review.

Without

codean agent builds against a spec still being edited; it shifts, and the work is built on a version that no longer exists.

uia screen is built from a mockup still in review; the mockup changes and the build is silently wrong.

default: enforced · independent-agent

A2Handover points

Defined transitions between stages, each a place a gate can attach. (A1 is which way work moves; this is that it moves in steps.)

Why: development moves in discrete steps with boundaries, not one fluid blur — the seams are where work can be stopped, inspected, and handed on.

With

deva plan is "done," a spec "approved," a change "ready" — defined seams you can pause at.

cieach pipeline stage is a gate a check can attach to (lint → test → build → deploy).

Without

devvibe-coding melts prompt → code → prompt into one stream with no seam to inspect or gate.

cione monolithic script does build+test+deploy with no checkpoint to stop or roll back at.

default: enforced · independent-agent

A3Intent locus

Humans own intent and values somewhere identifiable — a process with no human intent locus can't be held accountable.

Why: an accountable human owns the goal, so a wrong direction gets caught before it's built.

With

productevery feature traces to an accountable human owner; the "why" is a recorded decision.

researcha direction has a named human sponsor who can say "that's not what we're after."

Without

productagents optimize a proxy metric no human owns, and ship the wrong thing efficiently.

opsan automated system changes behavior with no human accountable for the intent behind it.

default: enforced · human (intent gate — never "nobody")

A4Ratifiable artifacts

Upstream can reach an approved state downstream consumes; outputs are checkable against it.

Why: you build against a stable, approved target with a real pass/fail criterion — not a moving one, and not "looks done to me."

With

speca spec reaches ratified, carries acceptance criteria, and work is graded against it.

dataa schema is versioned; downstream validates against the approved version, not HEAD.

Without

specnothing is ever "final," so implementation chases a spec that keeps moving under it.

designa design system has no "released" state, so teams build against inconsistent snapshots.

default: enforced · independent-agent

B Operating layer

What Trellis actually supplies once a process is admissible: the gates, the checks, the memory, and the discipline that keeps agents honest.

B1Graph maintenance

The dependency graph of artifacts and rules kept consistent and minimal; append-only records superseded, never edited in substance.

Why: the knowledge base stays coherent for the agents reading it, and a discovery deep in the code repairs the decision that should have known it (backprop).

With

docsa repaired decision re-reviews its specs → plans → code, in turn.

codea build uncovering a missing decision creates the record instead of patching around it.

researcha downstream finding that contradicts an upstream note updates the note, not just the finding.

Without

docsa decision changes but its dependent specs are never updated — they silently diverge.

dataa migration isn't propagated to the docs, so the docs now describe a DB that no longer exists.

rulesa stale lint rule keeps firing long after the pattern it guarded was removed.

default: enforced · independent-agent

B2Gate at handover

Apply the verification gate at every handover; any skip is surfaced.

Why: the review actually fires (not quietly skipped under deadline) — and if it is skipped, you can see it was.

With

cia conformance check + review fire on every PR; a deliberate skip is recorded, not hidden.

releasea promotion gate blocks an artifact that didn't pass the prior stage.

Without

devthe review is "optional," so under deadline it silently doesn't happen and a defect ships.

speca spec goes straight to implementation with no sign-off, and no record it was skipped.

default: default-on-but-skippable · independent-agent

B3Independent judgment

The assessor is independent of what it assesses — the builder doesn't grade itself, and the agent doesn't flatter the human.

Why: the builder doesn't grade its own homework, and the agent names the risk instead of flattering the plan — so verification and the intent gate are real, not decorative.

With

reviewa read-only reviewer, distinct from the author, derives its own checklist and reports what's wrong — even when inconvenient.

researchfindings are adversarially verified by a separate pass, not self-certified.

Without

codethe agent that wrote the code decides the code is good.

productthe agent agrees with a flawed plan to please the stakeholder, and it sails through.

default: default-on-but-skippable · independent-agent

B4Auditable archive

Provenance + immutable decision history + a consolidated current-truth.

Why: you can always answer "why is it this way?" — decisions are not lost or quietly rewritten.

With

adrdecisions are append-only and link their rationale; superseding writes a new record.

codegit history + a current-truth doc kept separate from its changelog.

Without

adra decision is edited in place and its rationale lost; months later it's re-litigated from scratch.

infraan undocumented prod change, so no one can say why it's configured this way.

default: enforced · independent-agent

B5Bounded context

Each operation reads only its declared inputs, never the whole archive.

Why: agents decide on sharp, relevant context instead of drowning in everything — better calls, and it scales as the archive grows.

With

agenta sub-agent gets exactly its declared inputs and decides crisply within them.

dataa query reads a scoped view, not the whole warehouse.

Without

agentan op dumps the entire repo into context, dilutes the signal, and decides on noise.

codea function reaches into global state instead of its parameters, and breaks when the state moves.

default: default-on-but-skippable · independent-agent

B6Self-improvement

The process learns from friction and gets better — improvement signals are surfaced and acted on, so a glitch doesn't happen twice.

Why: a process glitch never happens twice — friction becomes a fix, not a recurring tax.

With

cia flaky test recurs → a trigger is filed, the root cause fixed, the trigger retired in the same change.

processa repeated review miss becomes a checklist item that rides the PR you already write.

researcha recurring dead-end is recorded so the next inquiry skips it instead of re-walking it.

Without

cithe same pipeline step fails weekly and everyone just re-runs it, forever.

processa PR raises open questions with no follow-up and they rot, unowned.

onboardingthe same confusion hits every new contributor because no one wrote the lesson down.

default: default-on-but-skippable · human

B7Minimal first

The smallest process that works; add a step only when friction reveals the boundary; bias to retire over add.

Why: no ceremony for its own sake — every step has earned its place, so the process stays light.

With

processa deliberately tiny rule set; a new step lands only with a recorded reason.

toolinga build config with no unused steps; a dependency pulled in only when needed.

Without

processa heavyweight methodology copied wholesale, most steps cargo-culted.

codea whole framework pulled in to use one function.

default: expressed · human

B9Clarify before commit

Ambiguity in an upstream is surfaced and resolved (usually by asking a human) before downstream consumes it — never silently guessed.

Why: agents ask instead of guessing — you don't discover three files later that they took the wrong reading of a vague spec.

With

specan agent flags a vague requirement and resolves it before coding.

uia designer confirms an ambiguous interaction before building it.

Without

codean agent silently picks one reading of a vague spec, builds it, and it's the wrong one.

dataan ambiguous metric definition is guessed, and the dashboard is subtly, confidently wrong.

default: default-on-but-skippable · human

D The two floors

The only settings that never dial to zero. Everything else Trellis lets you tune; these two it holds firm.

D1Transparency

Every consequential choice is surfaced — a skipped gate, a missing capability, a degraded result, a divergence from a framework you claim to follow. Drift is allowed, but never silent.

Why: nothing consequential happens silently — you learn about the shortcut when it's taken, not when it breaks in production.

With

cia skipped gate is logged; a missing tool halts loudly, not silently.

framework"we diverge from Spec Kit here" is a recorded decision, so an agent knows where you follow the book and where you don't.

opsa degraded fallback (cache miss, retry, downgrade) is surfaced, not swallowed.

Without

codean agent silently falls back to a degraded path; you learn when it breaks in prod.

frameworkthe team quietly drifts from the methodology it claims to follow.

infraan error is swallowed and resurfaces later as a mystery outage.

floor: enforced (non-configurable to off) · human

D2The intent gate

The intent gate never fully opens — at the intent locus, a human (or a human-authorized independent check) is always mandatory. The one place an upstream that is itself wrong gets caught.

Why: "is this the right thing to build?" always has a human behind it — the one call you never hand fully to the machine.

With

producta human ratifies at the intent gate (here: the maintainer's merge).

releaseno deploy ships a feature no human approved, however green the pipeline.

Without

producta fully-automated pipeline ships something technically correct that no human confirmed was the right thing.

researchan agent auto-adopts a conclusion no human sponsored.

floor: enforced (non-configurable to off) · human (never "nobody")
